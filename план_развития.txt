=================================================================
План развития учебной VPN-инфраструктуры с использованием Yandex Cloud
=================================================================

1. Контейнеризация сервисов (2025 Q4)
-------------------------------------
1.1. Перенос компонентов в Docker:
    - Создать Docker-образы для:
        • Центра сертификации (Easy-RSA)
        • OpenVPN-сервера
        • Prometheus + Alertmanager

1.2. Управление контейнерами:
    - Развернуть Yandex Managed Service for Kubernetes (Yandex Kubernetes Engine)
    - Автоматизировать деплой через GitLab CI/CD:
        • Сборка образов при изменении кода
        • Автоматическое развертывание в кластере

2. Интеграция с сервисами Yandex Cloud
--------------------------------------
2.1. База данных для учета клиентов:
    - Внедрить Yandex Managed mySQL для хранения:
        • Учетных записей VPN-пользователей
        • Журнала выдачи сертификатов
        • Статистики подключений

2.2. Управление бэкапами:
    - Заменить самописные скрипты на Yandex Backup Service
    - Настроить автоматические снапшоты:
        • Ежедневные для БД mySQL
        • Еженедельные для конфигураций

3. Автоматизация инфраструктуры
-------------------------------
3.1. Инфраструктура как код (Terraform):
    - Создать модули для развертывания:
        • ВМ с предустановленным Docker
        • Managed mySQL
        • Kubernetes-кластера
    
3.2. CI/CD пайплайны:
    - Автоматическое тестирование конфигов при коммите
    - Провалидированные обновления через GitLab Runners

4. Мониторинг
-----------------------------
4.1. Улучшение оповещений:
    - Добавить интеграции:
        • Telegram-бот для критичных алертов
    

5. Оптимизация и масштабирование
--------------------------------
5.1. Serverless-функции:
    - Перенести генерацию клиентских конфигов в Yandex Cloud Functions
    - Триггеры:
        • По HTTP-запросу
        • По событию в БД
